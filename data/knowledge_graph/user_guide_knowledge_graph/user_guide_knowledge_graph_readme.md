# blastFoam 用户手册知识图谱说明

本文档旨在详细说明 `user_guide_knowledge_graph.json` 文件的结构和内容。该JSON文件是基于《blastFoam用户指导书》提取的知识图谱，旨在将手册内容结构化、机器可读化，便于后续的检索、分析和应用。

## 文件结构

`user_guide_knowledge_graph.json` 文件是一个包含多个JSON对象的数组。每个JSON对象代表知识图谱中的一个节点，对应于用户手册中的一个章节、一个概念、一个方程或一个具体的物理模型。

### 节点属性

知识图谱中的每个节点（JSON对象）都包含以下核心属性：

-   `id` (字符串): 节点的唯一标识符。通常对应于原始文档中的章节号（如 "ch2", "sec2.1"）或模型编号。
-   `parentId` (字符串 | null): 指向父节点的`id`。该属性构建了整个知识图谱的层级结构。如果值为`null`，表示这是一个顶层节点（通常是一个章节）。
-   `semantic_type` (字符串): 节点的语义类型。这是理解节点内容和功能的关键字段。
-   `title` (字符串): 节点所代表的章节或概念的标题（例如 "Governing equations", "Single-fluid model"）。
-   `number` (字符串): 在原始文档中的章节或模型编号。
-   `content_summary` (字符串): 对节点内容的简短总结，方便快速理解该节点的核心信息。
-   `content` (字符串): 节点的详细内容。其中包含了从原始文档中提取的文本和数学公式（以LaTeX格式表示）。
-   `table` (数组): 包含从文档中提取的表格数据。通常用于定义模型的参数和变量。数组中的每个对象代表表格的一行。

## 内容层级

通过 `id` 和 `parentId` 字段，整个知识图谱被组织成一个树状结构，清晰地反映了原始用户手册的章节和段落关系。

-   **顶层节点**: `parentId` 为 `null` 的节点是知识图谱的根节点，通常代表一个大的章节（`Chapter`），例如 `ch2` (Governing equations)。
-   **子节点**: 每个节点都可以通过其 `id` 作为其他节点的 `parentId`，从而形成父子关系。例如，`id` 为 `sec2.1` 的节点（Single-fluid model）的 `parentId` 是 `ch2`，表明它是“Governing equations”章节下的一个子部分。

这种层级结构使得遍历和理解文档的上下文关系变得非常容易。

## 语义类型 (semantic_type)

`semantic_type` 字段为每个节点赋予了明确的语义含义，是理解知识图谱内容的关键。主要的类型包括：

-   `ConceptCluster`: **概念集群**。这通常代表一个高级别的分组，如一个完整的章节（例如 `ch3: Interfacial models`）或一个包含多个相关模型的子章节（例如 `sec3.1: Drag models`）。它将一组相关的概念或模型组织在一起。

-   `GoverningEquation`: **控制方程**。这类节点描述了软件（blastFoam）所依据的核心物理方程。例如，`sec2.1: Single-fluid model` 和 `sec2.2: Multi-fluid` 详细定义了单流体和多流体模型所使用的守恒方程。

-   `PhysicalModel`: **物理模型**。这是知识图谱中最常见的类型之一，代表了在模拟中可以使用的具体模型。例如：
    -   `model3.1.1: SchillerNaumann`: 一个具体的曳力模型。
    -   `model5.2.2.8: JWL`: 一个用于含能材料的状态方程模型。
    -   `model5.6.1.2: linear`: 一个起爆激活模型。
    每个 `PhysicalModel` 节点都详细描述了该模型的用途、数学公式和相关参数。

## 内容与表格

-   **`content` 和 `content_summary`**: 这两个字段存储了从用户手册中提取的核心文本信息。`content_summary` 提供了简明扼要的概述，而 `content` 则包含完整的段落描述和使用LaTeX格式表示的复杂数学公式。这使得知识图谱不仅是结构化的，也保留了丰富的细节。

-   **`table`**: `table` 字段以结构化的数组形式存储了与模型相关的参数。例如，在 `model3.1.1: SchillerNaumann` 节点中，`table` 字段定义了 `residualRe` 这个参数及其描述。这对于自动化地提取模型参数和生成输入文件非常有用。

## 如何使用

这份知识图谱可以用于多种目的：

1.  **智能问答系统**: 作为后端知识库，用于回答关于blastFoam软件功能、模型选择和参数设置的问题。
2.  **代码/输入文件自动生成**: 通过解析 `PhysicalModel` 节点的参数表格，可以辅助用户或自动生成模拟所需的输入文件。
3.  **知识检索与学习**: 提供一个比传统PDF手册更易于导航和检索的结构化知识库，帮助新用户快速学习软件。

希望这份说明能帮助您更好地理解和使用 `user_guide_knowledge_graph.json` 文件。
