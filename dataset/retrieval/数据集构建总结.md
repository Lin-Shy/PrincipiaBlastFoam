## 数据集特点

### 1. 数据结构
每条记录包含：
```json
{
    "id": 唯一标识符,
    "query": "用户查询/修改请求",
    "description": "详细说明",
    "target_files": ["目标文件路径1", "目标文件路径2"],
    "difficulty": "难度等级",
    "category": "类别"
}
```

### 2. 覆盖范围广泛

#### 难度分布
- **基础级** (35条, 29%): 单文件简单修改
- **中级** (38条, 32%): 多参数或需要一定知识
- **高级** (47条, 39%): 复杂物理模型、多文件协调

#### 查询类型（4种）
1. **直接技术查询** (45条): "将 RASModel 从 kOmegaSST 改为 SpalartAllmaras"
2. **文件位置查询** (10条): "哪个文件控制湍流模型？"
3. **目标导向查询** (15条): "让模拟运行更长时间"
4. **复杂配置** (50条): 多参数、多文件修改

#### 类别分布（20种）
- 湍流模型 (3)
- 时间控制 (6)
- 网格 (9)
- 材料属性 (3)
- 燃烧 (7)
- 输出控制 (7)
- 初始条件 (7)
- 边界条件 (7)
- 状态方程 (3)
- 热力学 (2)
- 结构力学 (5)
- 数值格式 (4)
- 多相流 (10)
- 文件定位 (10)
- 目标导向 (15)

- 后处理 (4)
- 并行计算 (3)

### 3. 目标文件覆盖

最常见的目标文件：
1. `constant/phaseProperties` - 31次（材料、状态方程、多相流）
2. `system/controlDict` - 25次（时间、输出、求解器）
3. `constant/combustionProperties` - 12次（反应模型）
4. `constant/dynamicMeshDict` - 10次（自适应网格）
5. `system/setFieldsDict` - 9次（初始条件）
6. `constant/turbulenceProperties` - 8次（湍流）
7. `system/fvSchemes` - 6次（数值格式）
8. `0/U`, `0/p`, `0/T` - 边界条件
9. 其他配置文件

### 4. 实际应用场景

数据集涵盖真实使用场景：
- **参数研究**: 改变单一参数（时间、网格、密度）
- **模型对比**: 切换替代模型（湍流、状态方程）
- **网格细化**: 提高空间分辨率
- **稳定性问题**: 调整数值稳定性
- **精度提升**: 高阶格式、更细网格
- **物理增强**: 添加复杂性（多相、燃烧）
- **计算效率**: 并行设置、输出控制
- **问题排查**: 基于目标的问题解决

### 评估检索系统
```bash
python dataset/analyze_retrieval_dataset_simple.py --evaluate
```

评估指标：
- **精确匹配准确率**: 检索文件与目标完全匹配
- **平均精确度**: 检索文件中正确的比例
- **平均召回率**: 目标文件中被检索到的比例
- **F1分数**: 精确度和召回率的调和平均
- **平均倒数排名**: 第一个正确文件的排名倒数

## 集成到您的系统

### 1. 替换检索函数
修改 `analyze_retrieval_dataset_simple.py` 中的 `example_simple_retrieval_function`：

```python
def your_retrieval_function(query: str) -> List[str]:
    """您的检索实现"""
    # 调用您的 RAG 系统、语义搜索等
    retrieved_files = your_rag_system.retrieve(query)
    return retrieved_files
```

### 2. 运行评估
```python
from dataset.analyze_retrieval_dataset_simple import RetrievalEvaluator

evaluator = RetrievalEvaluator('dataset/retrieval/blastfoam_retrieval_validation_dataset.json')
metrics = evaluator.evaluate(your_retrieval_function)
evaluator.print_evaluation_results(metrics)
```
