# 数据集查询示例 - 按类别

本文档提供每个类别的典型查询示例，帮助理解数据集的实际应用场景。

注意：

- 本文件主要用于展示查询样式与任务类型，不是当前 strict 基准的权威标签说明。
- 文中很多“目标”仍采用相对文件路径或多文件示意，便于理解需求语义。
- 当前全 tutorial 严格评测请以 `benchmarks/case_content/blastfoam_retrieval_validation_dataset_strict.json` 和 `STRICT_RETRIEVAL_VALIDATION_README.md` 为准。

## 1. 基础级查询 (Basic)

### 时间控制 (time_control)
```
查询: "Increase the simulation end time to 0.005 seconds"
目标: system/controlDict
说明: 直接的时间参数修改
```

### 湍流模型 (turbulence_model)
```
查询: "Change the turbulence model from Spalart-Allmaras to kOmegaSST"
目标: constant/turbulenceProperties
说明: 切换湍流模型类型
```

### 网格 (mesh)
```
查询: "Adjust the mesh resolution to have 100 cells in x-direction, 50 in y, and 25 in z"
目标: system/blockMeshDict
说明: 修改基础网格分辨率
```

### 边界条件 (boundary_conditions)
```
查询: "Set initial velocity field to 500 m/s in x-direction for the moving cone"
目标: 0/U
说明: 设置速度边界条件
```

### 文件定位 (file_location)
```
查询: "What file controls the turbulence model selection?"
目标: constant/turbulenceProperties
说明: 用户询问哪个文件控制特定功能
```

## 2. 中级查询 (Intermediate)

### 多参数修改
```
查询: "Increase simulation time and reduce time step by setting maxCo to 0.2 and endTime to 0.01"

### 燃烧参数组合

```

### 输出控制组合
```

目标: system/controlDict

```
查询: "The simulation is unstable. How can I make it more stable?"

## 3. 高级查询 (Advanced)

### 状态方程切换
```
查询: "Change the equation of state for the explosive from perfectGas to JWL"
目标: constant/phaseProperties
说明: 切换到更复杂的 Jones-Wilkins-Lee 状态方程
```

### 燃烧模型升级
```
查询: "Switch combustion reaction model from laminar flame speed to Arrhenius kinetics"
目标: constant/combustionProperties
说明: 从简单模型切换到化学动力学模型
```

### 结构力学材料模型
```
查询: "Change window material model from linearElastic to elasticPlastic"
目标: constant/phaseProperties
说明: 启用塑性变形模拟
```

```
查询: "Change convection scheme from vanAlbada to MUSCL for higher accuracy"
目标: system/fvSchemes
说明: 切换到高阶数值格式
```

### 多相流模型
```
查询: "Use Gidaspow drag model instead of SchillerNaumann for particle-gas interaction"
目标: constant/phaseProperties

### 颗粒流动力学
```
查询: "Change granular viscosity model from Syamlal to HrenyaSinclair"

```

### 多文件物理模型切换
```
查询: "Change turbulence to kEpsilon and adjust combustion to use Arrhenius kinetics"
目标: constant/turbulenceProperties, constant/combustionProperties

### 目标导向 - 精度提升
```
查询: "Use a more accurate physics model for the explosive detonation"

```

## 5. 边界情况 (Edge Cases)

### 技术同义词
```
查询: "Use temperature-dependent gas properties for better accuracy"
目标: constant/phaseProperties
说明: "temperature-dependent" 应理解为需要 JANAF 表
```

### 隐式需求
```
查询: "Model turbulent flow effects in the explosion"
目标: constant/turbulenceProperties
说明: 需要从 laminar 切换到 RAS 模型
```

### 多种可能答案
```
查询: "Improve numerical accuracy"
目标: 可能是 system/fvSchemes 或 system/controlDict
说明: 可以通过高阶格式或更小时间步实现
```

## 6. 实际应用组合

### 爆炸模拟标准设置
```python
# 查询序列
1. "Change explosive properties: density to 1650 kg/m³ and use JWL equation of state"
   -> constant/phaseProperties

2. "Refine mesh to 200x100x50 cells and increase maxRefinement to 3"
   -> system/blockMeshDict, constant/dynamicMeshDict

3. "Set maxCo to 0.3 for stability"
   -> system/controlDict

4. "Setup window failure: pBurst 40 kPa and elasticPlastic material"
   -> constant/phaseProperties
```

### 燃烧模拟增强
```python
1. "Switch to kOmegaSST turbulence model"
   -> constant/turbulenceProperties

2. "Use Arrhenius reaction kinetics with A=1e8 and Ta=15000"
   -> constant/combustionProperties

3. "Set equivalence ratio to 0.9"
   -> constant/combustionProperties

4. "Use MUSCL scheme for better shock resolution"
   -> system/fvSchemes
```

### 颗粒爆炸模拟
```python
1. "Define three-phase system: air, explosive products, and solid particles"
   -> constant/phaseProperties

2. "Set particle properties: diameter 1.5 mm and use Gidaspow drag model"
   -> constant/phaseProperties

3. "Use HrenyaSinclair viscosity for particle phase"
   -> constant/turbulenceProperties.particles

4. "Initialize particle distribution in region (0.1, 0.3)"
   -> system/setFieldsDict
```

## 7. 按难度的推荐学习路径

### 初学者 (Basic 级别)
建议从这些类别开始：
1. file_location (了解文件结构)
2. time_control (简单参数修改)
3. output_control (输出设置)
4. boundary_conditions (边界条件基础)

### 进阶用户 (Intermediate 级别)
推荐练习：
1. mesh (网格生成和细化)
2. combustion (燃烧模型基础)
3. turbulence_model (湍流模型选择)
4. goal_based (目标导向问题解决)

### 专家用户 (Advanced 级别)
挑战性任务：
1. equation_of_state (复杂状态方程)
2. multiphase (多相流建模)
3. structural_mechanics (结构响应)
4. numerical_schemes (高级数值方法)
5. physics_models (多物理场耦合)

## 8. 常见错误模式

### 过度泛化
```
错误: "Change the simulation settings"
问题: 太模糊，无法确定具体文件
正确: "Change maxCo to 0.3 in controlDict"
```

### 混淆概念
```
错误: 将 "mesh refinement" 理解为只需要 blockMeshDict
正确: 可能需要 blockMeshDict 和/或 dynamicMeshDict
```

### 遗漏关联修改
```
错误: 只修改 turbulenceProperties 切换模型
正确: 还需要在 0/ 目录更新相应的湍流变量边界条件
```

## 9. 使用建议

### 开发检索系统时
1. 先处理 basic 级别的查询，确保基础功能正确
2. 逐步增加对 intermediate 级别的支持
3. 使用 advanced 级别测试系统上限
4. 特别关注 file_location 类别，这反映用户真实需求

### 评估系统性能时
1. 计算各难度级别的准确率
2. 识别低分类别，针对性改进
3. 关注多文件查询的召回率
4. 分析 goal_based 查询的理解能力

### 改进检索策略时
1. 建立技术术语词典
2. 处理同义词和缩写
3. 识别隐式需求
4. 考虑上下文依赖
