# 评估指标指导文档：自动化OpenFOAM案例生成的质量评估

本文档旨在详细阐述用于评估由AI自动生成的OpenFOAM案例质量的多维度评估指标。这些指标的设计基于 `experiments/end2end/validate_generated_cases.py` 脚本中的实现，旨在提供一个全面、可复现且自动化的评估框架。

## 1. 评估框架概述

为了全面评估AI生成案例的质量，我们设计了一个包含两个核心维度的评估体系。每个维度都从不同角度考察案例的有效性和可用性。最终，通过综合这些维度的结果，得出一个总体的质量评分。

该评估框架的核心优势在于：
- **多维度性**：结合了高级物理合理性和实际可执行性的多层次检查。
- **自动化**：整个评估流程由脚本自动执行，确保了客观性和可重复性。
- **深度验证**：利用大语言模型（LLM）和知识图谱对物理参数的合理性进行深度分析，这是传统方法难以实现的。

以下是两个核心评估维度：

1.  **物理参数合理性 (Physical Parameter Validity)**
2.  **案例可执行性 (Executability)**

## 2. 核心评估指标详解

### 2.1 物理参数合理性

- **目的 (Objective)**：利用大语言模型（LLM）和领域知识图谱，评估案例中设置的物理参数（如边界条件、材料属性、初始条件等）是否符合流体力学理论和工程实践。这是评估案例“质量”而非仅仅是“格式正确性”的关键步骤。
- **评估方法 (Methodology)**：
    1.  **上下文检索**：根据用户的原始需求（User Request），从OpenFOAM用户指南知识图谱中检索相关的理论和最佳实践文档。
    2.  **LLM分析**：构建一个详细的Prompt，将用户需求、修改后的文件内容、提取出的数值参数以及检索到的文档上下文全部提供给一个专家级的LLM。
    3.  **结构化判断**：LLM被要求扮演CFD专家的角色，对每个关键参数的物理合理性进行判断，并以结构化的JSON格式返回分析结果。分析内容包括：
        - 参数的物理意义和数值范围。
        - 计算无量纲数（如雷诺数、CFL数）以评估其合理性。
        - 与文档或理论相悖的设置。
        - 推荐的修正值或诊断方法。
- **评分标准 (Scoring)**：基于LLM返回的JSON中的 `status` 字段。
    - **Passed (通过)**：LLM认为所有参数设置均合理（`"status": "valid"`）。
    - **Warning (警告)**：LLM认为某些参数存在疑问但非致命错误（`"status": "questionable"`）。
    - **Failed (失败)**：LLM认为存在明显违反物理规律或求解器稳定性的严重问题（`"status": "invalid"`）。

### 2.2 案例可执行性

- **目的 (Objective)**：通过实际运行模拟来直接验证案例的最终可用性。这是检验案例质量最直接、最有效的“黄金标准”。
- **评估方法 (Methodology)**：
    1.  **执行脚本**：在案例目录下执行 `Allrun` 脚本，并设置一个合理的超时时间（例如300秒）。
    2.  **日志收集**：收集执行过程中产生的所有日志文件（如 `log.*`, `*.log` 等）。
    3.  **日志分析**：
        - **优先检查成功标志**：在日志的末尾部分查找 `End`、`solution converged`、`ExecutionTime =` 等明确表示计算完成或正常结束的标志。
        - **检查致命错误**：在整个日志中搜索 `FATAL`、`Segmentation fault`、`ERROR:` 等表示程序崩溃或严重错误的关键词。
- **评分标准 (Scoring)**：这是一个基于优先级的复杂判断：
    - **Passed (通过)**：日志中明确包含成功结束的标志，即使进程退出码非零。
    - **Failed (失败)**：日志中包含任何致命错误关键词。
    - **Warning (警告)**：
        - 进程因超时而中断，但日志中未发现致命错误（表明案例至少可以部分运行）。
        - 进程正常结束但未找到日志文件或成功标志。
        - 进程退出码非零，但日志中无明确错误。

## 3. 总体评估分数

为了得出一个量化的总体评估结果，我们将上述两个维度的评估状态进行综合。

- **计算方法 (Calculation)**：
    1.  首先，将每个维度的状态（`passed`, `warning`, `failed`）映射为一个分数（例如，`passed`=1.0, `warning`=0.5, `failed`=0.0）。
    2.  **分类别通过率 (Pass Rate by Category)**：每个维度的分数即代表该类别的通过率（乘以100%）。
    3.  **总体通过率 (Overall Pass Rate)**：计算所有四个维度分数的平均值，得到一个0到100%之间的总体通过率。
    4.  **总体状态 (Overall Status)**：
        - 如果任何一个维度为 `failed`，则总体状态为 `failed`。
        - 如果没有 `failed` 但至少有一个 `warning`，则总体状态为 `warning`。
        - 如果所有维度均为 `passed`，则总体状态为 `passed`。

- **报告与分析 (Reporting)**：
    最终的评估报告会展示每个案例的总体状态、总体通过率，以及各个细分维度的状态和通过率。通过对一批生成案例进行统计，我们可以计算出：
    - **平均总体通过率 (Average Overall Pass Rate)**：所有案例总体通过率的平均值。
    - **完全通过率 (Complete Pass Rate)**：总体状态为 `passed` 的案例占总数的百分比。
    - **各类别平均通过率 (Average Pass Rate by Category)**：分析在哪个维度上AI的表现最好或最差。

这份详尽的评估框架为我们衡量和改进AI在科学计算领域的应用提供了坚实的基础。
