# 基于知识图谱的分层检索方法技术文档

本文档详细介绍了在 `principia_ai` 项目中使用的基于 User Guide 知识图谱的分层检索方法。该方法旨在根据用户需求，高效、准确地从 BlastFoam 用户指南中检索相关信息。

## 1. 方法概述

该方法采用一种分层（Hierarchical）的检索策略，模拟人类查找文档的方式：首先定位到相关的章节，然后在这些章节中找到具体的节，最后在这些节中找到最相关的子节。这种方法通过逐步缩小搜索范围，提高了检索的准确性和效率。

整个流程分为以下几个主要步骤：

1.  **知识库加载**：加载简化的和完整的知识图谱。
2.  **第一层检索：识别相关章节**：根据用户查询，使用 LLM 识别最相关的文档章节。
3.  **第二层检索：识别相关节**：在已识别的章节中，进一步定位到相关的节。
4.  **第三层检索：识别相关子节**：在已识别的节中，递归地查找所有层级的子节，并确定最相关的子节。
5.  **内容检索**：获取最终确定的子节的完整内容。

## 2. 核心实现

核心实现位于 `principia_ai/tools/user_guide_knowledge_graph_tool.py` 文件中的 `UserGuideKnowledgeGraphRetriever` 类。

### 2.1. 知识库

系统使用了两种知识图谱：

*   **简化的知识图谱** (`simplified_user_guide_knowledge_graph.json`)：包含章节和节的标题、编号和内容摘要。用于前两层的快速检索，以减少 LLM 的处理负担。
*   **完整的知识图谱** (`user_guide_knowledge_graph.json`)：包含所有节点（章节、节、子节）的完整信息，包括详细内容和表格数据。用于最后一层的内容检索。

两种知识图谱都在 `_load_knowledge_bases` 方法中加载。

### 2.2. 分层检索流程

#### 步骤 1: 识别相关章节 (`_identify_relevant_chapters`)

-   **输入**：用户查询 (`query`)。
-   **过程**：
    1.  从简化的知识图谱中提取所有章节级别（ID 以 `ch` 开头）的节点。
    2.  构建一个包含所有章节标题和编号的列表。
    3.  将用户查询和章节列表组合成一个 Prompt，要求 LLM 以 JSON 格式返回相关的章节 ID。
-   **输出**：一个包含相关章节 ID 的列表 (例如, `['ch2', 'ch19']`)。

#### 步骤 2: 识别相关节 (`_identify_relevant_sections`)

-   **输入**：用户查询 (`query`) 和第一步识别出的章节 ID 列表 (`chapter_ids`)。
-   **过程**：
    1.  从简化的知识图谱中，筛选出所有父节点属于 `chapter_ids` 的节。
    2.  构建一个包含这些节的编号、标题和内容摘要的列表。
    3.  将用户查询和节列表组合成一个 Prompt，要求 LLM 以 JSON 格式返回最相关的节 ID。
-   **输出**：一个包含相关节 ID 的列表 (例如, `['sec2.1', 'sec5.2']`)。如果此步骤没有找到相关的节，将返回上一层的章节 ID 作为后备。

#### 步骤 3: 识别相关子节 (`_identify_relevant_subsections`)

-   **输入**：用户查询 (`query`) 和第二步识别出的节 ID 列表 (`section_ids`)。
-   **过程**：
    1.  这是一个递归过程。首先，将 `section_ids` 自身包含的节点加入搜索范围。
    2.  然后，从完整的知识图谱中，递归地查找所有隶属于 `section_ids` 的子节（孙子节、曾孙子节等）。
    3.  构建一个包含所有这些节和子节的编号、标题和内容摘要的列表。
    4.  将用户查询和这个列表组合成一个 Prompt，要求 LLM 以 JSON 格式返回最相关的子节**编号**。
    5.  将返回的子节编号映射回对应的节点 ID。
-   **输出**：一个包含最具体的相关子节 ID 的列表。如果此步骤失败，将返回上一层的节 ID 作为后备。

#### 步骤 4: 检索完整内容 (`_retrieve_full_content`)

-   **输入**：第三步识别出的子节 ID 列表 (`subsection_ids`)。
-   **过程**：
    1.  遍历 `subsection_ids`。
    2.  对于每个 ID，从完整的知识图谱中查找对应的节点。
    3.  格式化节点的标题、摘要、内容和表格信息。
-   **输出**：一个包含所有检索到的格式化内容的字符串。

### 2.3. 主搜索方法 (`search`)

`search` 方法是整个流程的入口点，它按顺序调用上述四个步骤，并将最终检索到的内容返回。

## 3. 使用场景

该分层检索方法主要用于需要从大量结构化文档中精确查找信息的场景。在本项目中，它被用于 `CaseValidator` 中，以验证生成的 OpenFOAM 算例中的物理参数是否合理。

在 `_validate_physical_parameters` 方法中，`UserGuideKnowledgeGraphRetriever` 被调用，以获取与用户请求相关的文档上下文。然后，这个上下文与用户的请求和修改的文件内容一起被发送给 LLM，以进行物理合理性分析。

## 4. 总结

基于知识图谱的分层检索方法通过模拟人类专家的查阅习惯，实现了高效、精准的知识检索。它将一个复杂的检索任务分解为多个简单的、逐步深入的子任务，有效地利用了 LLM 的能力，同时避免了将大量无关信息一次性提供给 LLM 所带来的性能和成本问题。这种分层设计使得检索过程更加可控和可解释。
