# 用户手册知识检索技术文档

## 1. 概述
`UserGuideKnowledgeGraphRetriever` 是本项目中用于从 BlastFoam 用户手册中检索信息的专用工具。它采用**分层检索（Hierarchical Retrieval）**策略，通过逐步缩小搜索范围（从章节到小节，再到具体内容），结合 LLM 的语义理解能力，实现高精度的文档内容定位。

该方法特别适合处理长文档检索任务，避免了直接将大量文档内容塞入 LLM 上下文窗口（Context Window）导致的精度下降和成本增加问题。

## 2. 核心架构

### 2.1 双知识库设计
为了平衡检索效率和内容完整性，该工具使用了两个版本的知识库文件：

1.  **简化知识库 (`simplified_user_guide_knowledge_graph.json`)**:
    - **内容**: 仅包含文档的结构信息（章节号、标题、层级关系）和内容摘要（Summary）。
    - **用途**: 用于检索的前几个阶段（章节识别、小节识别），减少 LLM 的 Token 消耗，提高处理速度。
2.  **完整知识库 (`user_guide_knowledge_graph.json`)**:
    - **内容**: 包含文档节点的完整文本内容。
    - **用途**: 仅在最终阶段使用，用于提取并返回用户所需的具体信息。

### 2.2 检索流程
检索过程被设计为一个漏斗形的四步流程：

#### 第一步：章节识别 (Chapter Identification)
- **方法**: `_identify_relevant_chapters`
- **输入**: 用户查询字符串。
- **逻辑**: 
    1. 从简化库中筛选出所有顶层章节节点（ID 以 'ch' 开头）。
    2. 构建 Prompt，包含章节号和标题。
    3. LLM 判断哪些章节可能包含相关信息。
- **输出**: 相关章节 ID 列表 (例如: `['ch2', 'ch5']`)。

#### 第二步：小节识别 (Section Identification)
- **方法**: `_identify_relevant_sections`
- **输入**: 用户查询 + 第一步识别出的章节 ID。
- **逻辑**: 
    1. 获取选中章节下的所有一级小节。
    2. 构建 Prompt，包含小节标题和内容摘要。
    3. LLM 进行二次筛选。
- **输出**: 相关小节 ID 列表。

#### 第三步：子小节识别 (Subsection Identification)
- **方法**: `_identify_relevant_subsections`
- **输入**: 用户查询 + 第二步识别出的小节 ID。
- **逻辑**: 
    1. **递归获取**: 获取选中小节下的所有子孙节点（无论层级深浅），确保不遗漏深层细节。
    2. **上下文包含**: 搜索范围同时包含父级小节本身。
    3. **LLM 决策**: LLM 根据具体的子小节标题和摘要，锁定最精确的文档片段。
- **输出**: 目标子小节 ID 列表。

#### 第四步：内容获取 (Content Retrieval)
- **方法**: `_retrieve_full_content`
- **输入**: 目标节点 ID 列表。
- **逻辑**: 
    1. 使用 ID 从**完整知识库**中查找对应的节点。
    2. 提取 `content` 字段（完整文本）。
    3. 格式化拼接并返回最终结果。

## 3. 代码实现细节

### 类结构 (`UserGuideKnowledgeGraphRetriever`)
- **初始化 (`__init__`)**: 
    - 初始化 LLM 客户端 (ChatOpenAI)。
    - 加载两个 JSON 知识库到内存。
    - 构建 `id_to_node` 字典以加速查找。
- **主入口 (`search`)**: 
    - 串联上述四个步骤，将每一步的输出作为下一步的输入。

### 关键技术点
- **ID 映射优化**: 使用字典哈希表实现 O(1) 复杂度的节点查找，避免遍历列表。
- **递归搜索**: `get_all_children` 辅助函数用于获取某节点下的完整子树，这是处理多层级文档结构的关键。
- **ID 归一化**: `_normalize_node_id` 方法用于处理 LLM 输出可能存在的格式差异（如将 "Section 2.1" 转换为系统内部 ID "sec2.1"），增强了系统的鲁棒性。
- **Prompt 工程**: 在每一步的 Prompt 中，通过 JSON Mode (`response_format={"type": "json_object"}`) 强制 LLM 输出结构化数据，便于程序解析。

## 4. 数据结构示例
知识图谱中的节点通常包含以下字段：
```json
{
  "id": "sec2.1.3",
  "title": "Time Integration Schemes",
  "number": "2.1.3",
  "parentId": "sec2.1",
  "content": "The RK4 scheme is implemented as...",  // 仅在完整库中
  "content_summary": "Details about RK4 implementation..." // 在简化库中
}
```
